# Self-checklist проекта

Чек-лист самопроверки перед сдачей проекта.

## 1. Постановка задачи

- [x] Описана прикладная задача прогнозирования временного ряда.
- [x] Определён пользователь системы: аналитик/исследователь/инженер.
- [x] Описан вход системы: исторические наблюдения и/или сетка прогноза.
- [x] Описан выход системы: прогноз и дисперсия прогноза.
- [x] Сформулированы метрики качества: MAE, RMSE, MAPE, sMAPE.

## 2. Данные

- [x] Используется реальный открытый датасет SILSO.
- [x] Исходный файл лежит в `project/data/raw/`.
- [x] Обработанный датасет сохраняется в `project/data/processed/`.
- [x] Отфильтрованы некорректные значения `SSn < 0`.
- [x] Используется time-based split без перемешивания.
- [x] В проекте нет персональных данных.

## 3. Модели и эксперименты

- [x] Реализована основная модель `IRFKriging`.
- [x] Используется kernel `RationalQuadratic + Periodic + Nugget`.
- [x] Модель обучается через `train.py`.
- [x] Обученная модель сохраняется в `project/artifacts/model.pkl`.
- [x] Метрики сохраняются в `project/artifacts/metrics.json`.
- [x] Реализован отдельный `predict.py`.
- [x] Реализовано сравнение с baseline-моделями.
- [x] Baseline-результаты сохраняются в CSV/JSON.
